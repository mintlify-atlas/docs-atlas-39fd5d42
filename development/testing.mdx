The Muun Wallet iOS app includes comprehensive unit tests and UI automation tests to ensure reliability and security.

## Test Targets

The project includes two test targets:

<Steps>
  <Step title="falconTests (Unit Tests)">
    Tests business logic, data layer, and domain layer components.
    
    **Location:** `falcon/app/falconTests/`
    
    **Test count:** 100+ unit tests
  </Step>

  <Step title="falconUITests (UI Tests)">
    End-to-end UI automation tests using XCTest UI testing framework.
    
    **Location:** `falcon/app/falconUITests/`
  </Step>
</Steps>

## Running Tests

### Run Tests in Xcode

<Steps>
  <Step title="Open the workspace">
    ```bash
    cd falcon/app
    open falcon.xcworkspace
    ```
  </Step>

  <Step title="Select test scheme">
    Choose the **Muun** scheme in the scheme selector
  </Step>

  <Step title="Run tests">
    - Press **Cmd+U** to run all tests
    - Press **Ctrl+Opt+Cmd+U** to run tests with code coverage
    - Click the diamond icon next to a test method to run a single test
  </Step>
</Steps>

<CodeGroup>

</CodeGroup>

## Base Test Class

All unit tests inherit from `MuunTestCase`, which sets up the testing environment:

```swift falconTests/MuunTestCase.swift
import XCTest
import Dip
import GRDB
import RxSwift
import RxBlocking
@testable import Muun
import Libwallet

class MuunTestCase: XCTestCase {
    var containers: [DependencyContainer] = []
    var userDefaults: UserDefaults!
    var compositeDisposable: CompositeDisposable?
    
    override func setUp() {
        try? FileManager.default.removeItem(at: libwalletStorageURL)
        
        setupTestingContext()
        compositeDisposable = CompositeDisposable()
        
        // Clean database before each test
        try! (resolve() as DatabaseCoordinator).wipeAll()
    }
    
    override func tearDown() {
        // Clean up after each test
        (resolve() as SecureStorage).wipeAll()
        try! (resolve() as DatabaseCoordinator).wipeAll()
        try? FileManager.default.removeItem(at: libwalletStorageURL)
        
        // Reset dependency containers
        DIManager.testing_resetCurrentContainer()
        containers.forEach { $0.reset() }
        containers = []
        
        compositeDisposable?.dispose()
        userDefaults.removePersistentDomain(forName: userDefaultsDomain)
    }
}
```

<Note>
  The base test class:
  - Creates isolated test databases
  - Sets up dependency injection containers
  - Cleans up all state between tests
  - Provides helper methods for async testing
</Note>

### Testing Repositories

```swift falconTests/Repositories/OperationRepositoryTests.swift
import XCTest
import RxSwift
import RxBlocking
@testable import Muun

class OperationRepositoryTests: MuunTestCase {
    var repository: OperationRepository!
    
    override func setUp() {
        super.setUp()
        repository = resolve()
    }
    
    func test_storeAndFetch() throws {
        // Given
        let operation = Factory.createOperation()
        
        // When
        try repository.store(operation)
        let fetched = try repository.fetch(by: operation.id).toBlocking().first()!
        
        // Then
        XCTAssertEqual(fetched?.id, operation.id)
        XCTAssertEqual(fetched?.amount, operation.amount)
    }
    
    func test_watchOperations() throws {
        // Test reactive observation of operations
        let expectation = self.expectation(description: "operations changed")
        
        let disposable = repository.watchOperations()
            .subscribe(onNext: { operations in
                if operations.count == 1 {
                    expectation.fulfill()
                }
            })
        
        // Trigger change
        try repository.store(Factory.createOperation())
        
        waitForExpectations(timeout: 5.0)
        disposable.dispose()
    }
}
```

## UI Testing with Page Objects

UI tests use the Page Object pattern for maintainability:

```swift falconUITests/Presentation/TransactionListPage.swift
import XCTest

class TransactionListPage: BasePage {
    private let transactionCell = XCUIApplication().cells["transaction_cell"]
    private let emptyStateView = XCUIApplication().staticTexts["empty_state"]
    
    func tapTransaction(at index: Int) {
        transactionCell.children(matching: .cell).element(boundBy: index).tap()
    }
    
    func assertHasTransactions() {
        XCTAssertTrue(transactionCell.exists)
    }
    
    func assertEmpty() {
        XCTAssertTrue(emptyStateView.exists)
    }
}
```

## Best Practices

<Steps>
  <Step title="Isolate tests">
    Each test should be independent and not rely on the state from other tests. The `MuunTestCase` base class ensures clean state.
  </Step>

  <Step title="Use factories">
    Create test data using factory methods for consistency and maintainability.
  </Step>

  <Step title="Test behavior, not implementation">
    Focus on testing what the code does, not how it does it.
  </Step>

  <Step title="Mock external dependencies">
    Use mock objects for network services, secure storage, and other external systems.
  </Step>

  <Step title="Keep tests fast">
    Unit tests should run in milliseconds. Avoid unnecessary waits and network calls.
  </Step>
</Steps>

## Troubleshooting Tests

<Accordion title="Tests fail due to libwallet errors">
  Ensure libwallet is built correctly:
  ```bash
  cd libwallet
  ../tools/libwallet-ios.sh
  ```
</Accordion>

<Accordion title="Database errors in tests">
  The test database might be corrupted:
  ```bash
  rm -rf ~/Library/Developer/CoreSimulator/Devices/*/data/Library/ApplicationSupport/test-db.sqlite
  ```
</Accordion>

<Accordion title="UI tests fail to launch app">
  1. Reset simulator: **Device > Erase All Content and Settings**
  2. Ensure the app scheme is set to **Shared**
  3. Check that UI testing arguments are configured
</Accordion>

<Accordion title="Flaky async tests">
  Increase timeout values or use proper expectations:
  ```swift
  let expectation = self.expectation(description: "async operation")
  // ... perform async work ...
  waitForExpectations(timeout: 10.0)
  ```
</Accordion>
