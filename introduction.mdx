# Welcome to Muun Wallet

Muun is a **non-custodial 2-of-2 multisig Bitcoin and Lightning wallet** for iOS, designed with a focus on security and ease of use. This documentation covers the iOS application architecture, the underlying libwallet core library, and guides for developers contributing to the project.

## Architecture Overview

Muun Wallet is built with three main components:

### iOS Application (Swift)

The iOS app follows clean architecture principles with three distinct layers:

- **Data Layer** - Handles database, keychain, secure storage, and network operations
- **Domain Layer** - Contains business logic, models, and use cases
- **Presentation Layer** - Manages UI components and user interactions

### Libwallet (Go)

The core cryptographic and Bitcoin-specific logic is written in Go and bridged to iOS using gomobile:

- Address generation and derivation
- Transaction creation and signing
- Fee estimation
- Lightning Network operations
- Emergency Kit generation and recovery
- MuSig2 cryptographic operations

### Prover (Rust)

Advanced cryptographic proofs using plonky2 for zero-knowledge validation.

## Security Model

Muun uses a **2-of-2 multisig** setup where:

1. **User key** - Generated and stored securely on your device
2. **Server key** - Held by Muun's cosigning server

Both keys are required to sign transactions, providing:

- Protection against server compromise (server cannot spend your funds alone)
- Recovery options via the Emergency Kit if the server becomes unavailable
- No single point of failure

<Note>
The Emergency Kit allows you to recover your funds unilaterally using only your device and recovery code, even if Muun's servers are permanently offline.
</Note>

## Non-Custodial Design

Muun is **non-custodial**, meaning:

- You control your private keys
- Muun cannot access or move your funds without your participation
- Your funds remain yours at all times
- The Emergency Kit provides a fallback recovery mechanism
