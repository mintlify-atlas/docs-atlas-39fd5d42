This guide will walk you through cloning the repository and getting the Muun Wallet iOS app running on your machine.

## Prerequisites

Before you begin, ensure you have the following installed:

- macOS (required for iOS development)
- Xcode 14.0 or later with Command Line Tools
- [Go](https://golang.org/) 1.19 or later
- [CocoaPods](https://cocoapods.org/)
- [Docker](https://docs.docker.com/engine/install/)
- Git

<Note>
  Make sure your `GOPATH` environment variable is properly configured. You can verify this by running `echo $GOPATH` in your terminal.
</Note>

## Clone the Repository

<Steps>
  <Step title="Clone the source code">
    ```bash
    git clone https://github.com/muun/falcon.git
    cd falcon
    ```
  </Step>

  <Step title="Build the Rust libraries">
    The Muun wallet uses Rust libraries for cryptographic operations. Build them with Docker:

    ```bash
    ./libwallet/librs/makelibs.sh
    ```

    <Info>
      This script builds the Rust bindings for iOS targets (aarch64-apple-ios, x86_64-apple-ios, aarch64-apple-ios-sim) and creates an .xcframework bundle. The build process uses Docker to ensure reproducible builds.
    </Info>

    The script will:
    - Build a Docker image with necessary Rust toolchains
    - Generate proof/verifier data for runtime verification
    - Compile bindings for all iOS architectures
    - Create `libs/librs.xcframework` using `lipo` and `xcodebuild`
  </Step>

  <Step title="Bootstrap gomobile">
    Install and configure gomobile to bridge the Go libwallet to iOS:

    ```bash
    ./tools/bootstrap-gomobile.sh
    ```

    This installs:
    - `golang.org/x/mobile/cmd/gomobile`
    - `golang.org/x/mobile/cmd/gobind`

    The script uses a shared dependency cache in `libwallet/.build/pkg` to speed up subsequent builds.
  </Step>

  <Step title="Install CocoaPods dependencies">
    Navigate to the iOS app directory and install dependencies:

    ```bash
    cd falcon/app
    pod install
    ```

    <Warning>
      During the `pre_install` phase, CocoaPods will automatically:
      - Save your GOPATH to `.gopath` file (needed for Xcode builds)
      - Run `tools/libwallet-ios.sh` to build the libwallet.xcframework
      
      This step may take 10-15 minutes on first run.
    </Warning>

    Key dependencies being installed:
    - Firebase (Core, Messaging, Analytics, Crashlytics)
    - GoogleSignIn for Drive backups
    - GRDB.swift for local database
    - RxSwift for reactive programming
  </Step>

  <Step title="Open the workspace in Xcode">
    ```bash
    open falcon.xcodeproj
    ```

    <Note>
      Always open the `.xcodeproj` file (or `.xcworkspace` if it exists after pod install), not individual `.xcodeproj` files. This ensures CocoaPods dependencies are properly linked.
    </Note>
  </Step>

  <Step title="Select a development team">
    In Xcode:
    1. Select the project in the navigator
    2. Go to **Signing & Capabilities**
    3. Select your development team
    4. Choose a simulator or connected device
  </Step>

  <Step title="Build and run">
    Press `Cmd + R` or click the Run button in Xcode.

    <Info>
      The first build compiles the libwallet Go code into an iOS framework. Subsequent builds will skip this step if the source hasn't changed (verified via SHA checksums).
    </Info>
  </Step>
</Steps>

## Verify Your Setup

Once the app launches, you should see the Muun Wallet welcome screen. You can:

- Create a new wallet
- Recover an existing wallet
- Explore the app in the simulator

<Note>
  For development, the app may use localhost connections. Check `tools/ios-assign-localhost-ip-for-debug.sh` if you need to configure debug networking.
</Note>

## Project Architecture

The Muun iOS app follows a clean architecture pattern with three main layers:

- **Data Layer** (`falcon/app/falcon/Data/`): Handles databases, OS interactions, and network calls
- **Domain Layer** (`falcon/app/falcon/Domain/`): Contains business logic and use cases
- **Presentation Layer** (`falcon/app/falcon/Presentation/`): UI code and view controllers
- **libwallet** (`libwallet/`): Go-based crypto and Bitcoin operations (bridged via gomobile)

## Common Issues

<AccordionGroup>
  <Accordion title="gomobile: command not found">
    Ensure you've run `./tools/bootstrap-gomobile.sh` and that `$GOPATH/bin` is in your PATH:

    ```bash
    export PATH="$GOPATH/bin:$PATH"
    ```
  </Accordion>

  <Accordion title="Docker permission denied">
    On Linux, you may need to add your user to the docker group:

    ```bash
    sudo usermod -aG docker $USER
    ```

    Then log out and back in.
  </Accordion>

  <Accordion title="Xcode Command Line Tools not found">
    Install or update Command Line Tools:

    ```bash
    xcode-select --install
    ```

    Verify in Xcode > Preferences > Locations > Command Line Tools.
  </Accordion>

  <Accordion title="libwallet.xcframework build fails">
    Check that:
    1. Docker is running
    2. Go version is 1.19+: `go version`
    3. GOPATH is set: `echo $GOPATH`
    4. You have internet access (gomobile downloads dependencies)

    Try cleaning and rebuilding:

    ```bash
    rm -rf libwallet/.build
    rm -rf falcon/core/Libwallet.xcframework
    cd falcon/app && pod install
    ```
  </Accordion>
</AccordionGroup>
